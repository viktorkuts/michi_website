# synq-web

Marketing site for **Synq** — a real-life social app. Tagline: *More life
outside.* Built per the contract in the master build prompt; design system
locked in [DESIGN_BRIEF.md](./DESIGN_BRIEF.md).

## Stack

- Nuxt 4 + Vue 3.5 + TypeScript (strict)
- Tailwind CSS v4 (CSS-first config) + token-driven CSS custom properties
- @nuxt/content v3 (legal pages + FAQ)
- @nuxt/image (AVIF/WebP)
- motion-v (Vue port of Motion / framer-motion)
- GSAP + ScrollTrigger (hero scrub + sticky pin only)
- @vueuse/core
- Self-hosted Geist Sans + Geist Mono

## Setup

```bash
npm install
```

The `postinstall` step runs `nuxt prepare` and copies Geist woff2 files
from `node_modules/geist/` into `public/fonts/`.

## Development

```bash
npm run dev          # http://localhost:3000
npm run typecheck    # tsc --noEmit through nuxi
```

## Production

```bash
npm run build        # Nuxt SSR build
npm run preview      # Preview the production build locally
npm run generate     # Static export
```

## Project structure

```
assets/
  css/
    tokens.css           Design tokens — DO NOT bypass these
    typography.css       Geist @font-face + .type-* utility classes
    base.css             Resets, container, section shell, focus rings
components/
  ui/                    Primitives (UiButton, UiNavLink, ...)
  layout/                LayoutNav, LayoutFooter, LayoutShell, ...
  section/               One file per homepage section
composables/
  useFrameSequence.ts    Hero canvas frame preloader + scrubber
  useNavState.ts         Nav transparent / frosted / idle state machine
  useActiveSection.ts    IntersectionObserver-based nav indicator
  useReducedMotion.ts    prefers-reduced-motion wrapper
  useMarquee.ts          Horizontal-scroll marquee logic
  useFaq.ts              FAQ tab + accordion state
content/
  legal/                 privacy.md, terms.md, account-deletion.md
  faq/                   getting-started.md, safety.md, account.md, plans.md
  gallery-tags.json      12 entries — image filename + caption tag
  about-tags.json        About-section image tags
  stats.json             Stats with sources and verify flags
pages/
  index.vue              Single-page scroll experience
  privacy.vue            renders content/legal/privacy.md
  terms.vue              renders content/legal/terms.md
  account-deletion.vue   renders content/legal/account-deletion.md
  contact.vue            Contact form / mailto fallback
public/
  hero/desktop|mobile/   90 frames each as WebP (poster fallback included)
  gallery/               12 portrait images for the marquee
  about/                 anchor + 3-5 satellite images
  fonts/                 Geist Sans/Mono woff2 (generated by setup-fonts.mjs)
scripts/
  setup-fonts.mjs        Copies Geist files from node_modules into public/fonts
  setup-hero-frames.mjs  Generates placeholder hero frames if none exist
server/
  api/                   Contact form endpoint (optional)
```

## The contract

- [DESIGN_BRIEF.md](./DESIGN_BRIEF.md) — locked design system. All
  components reference its tokens. Edits to tokens go here first.
- [SKILL_AUDIT.md](./SKILL_AUDIT.md) — proof of the 8 skill reads required
  by the master build prompt.
- [CONTENT.md](./CONTENT.md) — every placeholder copy block in the site,
  flagged for user review.
- [CONSISTENCY_CHECKLIST.md](./CONSISTENCY_CHECKLIST.md) — per-section QA.

## Brand-color rule

`#6765EC` is the only chromatic color. Used for: primary CTA fills, focus
rings, active nav dot, link hover underlines, the about-section signature,
the loader bar. Never for text on ivory, never as a background fill, never
as a gradient. Budget: ≤5% of any viewport's pixel area.

## Performance budget

- LCP < 2.5s on simulated 4G
- Initial JS gzipped < 180KB
- Hero frame sequence < 1.5MB
- Lighthouse Performance ≥ 90 mobile
- CLS < 0.05
